# 찝찝함 번역기 디자인 설계

작성일: 2026-05-15

## 배경

찝찝함 번역기는 관계를 끊고 싶지는 않지만 지나친 친밀감과 감정 노동은 피하고 싶은 사용자를 위한 AI 기반 웹앱이다. 핵심 가치는 상대의 속마음을 맞히는 것이 아니라, 사용자가 느낀 찝찝함을 감정, 사실, 해석, 대응 문장으로 나누어 차분하게 볼 수 있게 돕는 것이다.

PRD v2에서 추가된 경쟁 앱 분석에 따라, 제품은 Pause의 "보내기 전 잠깐 멈춤" 경험을 참고하되 감정분석 앱처럼 얼굴, 음성, 영상 또는 속마음 판독으로 확장하지 않는다. 결과는 직관적이어야 하지만 퍼센트나 진단처럼 보이는 표현은 피한다.

## 승인된 UI 방향

전체 무드는 `Social Tool` 방향을 사용한다. 밝은 화이트 기반, 선명한 인디고와 오렌지 포인트, 짧고 앱다운 인터랙션을 중심으로 한다. 전문성은 과도한 무게감이 아니라 보안 모드, Trial 상태, 비단정적 결과 구조, 명확한 데이터 처리 문구에서 만든다.

첫 화면 구조는 `Input First`로 확정한다. 사용자가 서비스 설명을 오래 읽기보다 바로 자기 상황이나 보내려는 문장을 입력하고 첫 분석까지 도달하는 흐름을 우선한다. 6개 메뉴는 별도 랜딩 영역이 아니라 입력창 주변의 탭 또는 세그먼트 컨트롤로 노출한다.

## 첫 화면

첫 화면의 주 메시지는 "지금 보내려는 말, 한 번만 정리해볼까요?"이다. 이는 단순한 답장 생성보다 "잠깐 멈추고 정리하기"라는 제품 태도를 보여준다.

주요 요소:

- 상단 브랜드명: 찝찝함 번역기
- 상태 배지: Trial 7일, 저장 안 함
- 신뢰 안내: 상대 속마음을 맞히지 않고, 내 감정과 확인 가능한 사실부터 분리한다
- 메뉴 탭: 답장 온도, 선 넘음, 세대차이, 거리두기, 관계 유지, 피로도
- 입력 영역: 상황 또는 보내려는 문장
- 선택 필드: 관계 유형, 원하는 도움, 톤
- 주요 CTA: 10초 멈추고 정리하기

## 결과 화면

결과 화면은 사용자가 AI 판단을 정답처럼 받아들이지 않도록 가능성 중심으로 구성한다. 퍼센트 대신 라벨, 단계, 카드형 구분을 사용한다.

기본 결과 구조:

- 상황 요약
- 감정 / 사실 / 해석 분리
- 상대 말의 가능한 해석
- 과해석 가능성
- 대응 필요도
- 말투 리스크
- 보내도 되는 문장 3개
- 피해야 할 표현
- 다음 행동 제안

결과 카드에는 `진단 아님`, `복사 가능`, `저장 안 함` 같은 작은 상태값을 함께 보여준다. 사용자가 바로 쓸 수 있는 문장에는 복사 버튼을 제공하고, "조금 부드럽게", "더 단호하게", "직장용으로" 같은 재작성 액션을 제공한다.

## 데이터와 안전

기본값은 저장하지 않는 보안 모드다. 입력한 내용은 결과 생성 후 폐기한다는 문구를 첫 화면 입력창 주변에 둔다. 사용자가 저장을 선택한 경우에만 결과 저장을 허용한다.

AI 응답은 다음 원칙을 따른다:

- 상대 의도를 단정하지 않는다.
- 사용자의 편만 들어주지 않는다.
- 가능성과 확인 가능한 사실을 분리한다.
- 치료, 진단, 법률 자문처럼 보이지 않게 한다.
- 폭력, 괴롭힘, 자해, 스토킹 등 위험 신호는 공식 도움 요청 안내를 우선한다.

## MVP 범위

첫 구현은 웹앱 MVP로 진행한다.

포함:

- 랜딩 없이 바로 사용 가능한 첫 화면
- 6개 메뉴 탭
- 상황 입력 폼
- 관계 유형 선택
- 원하는 도움 선택
- 톤 선택
- AI 결과 카드 UI
- 문장 단위 복사
- Trial 안내와 사용량 표시
- 보안 모드 안내

제외:

- 모바일 네이티브 앱
- 장기 관계 히스토리
- 결제 연동
- B2B 관리자 기능
- 강사용 워크시트 자동 생성
- 얼굴, 음성, 영상 분석

## 구현 방향

기존 프로젝트는 Next.js와 Tailwind CSS 기반이므로 해당 구조를 유지한다. 첫 단계에서는 서버 연동보다 UI와 입력/결과 상태 흐름을 우선 구현한다. 실제 AI API 연동은 결과 구조와 프롬프트 계약이 안정된 후 붙인다.

주요 컴포넌트 경계:

- 앱 첫 화면: 전체 입력과 결과 상태를 조율한다.
- 메뉴 탭: 분석 모드를 선택한다.
- 입력 패널: 상황, 관계, 도움, 톤을 수집한다.
- 결과 패널: 구조화된 결과 카드를 렌더링한다.
- 보안 안내: 저장 안 함, 진단 아님, Trial 정보를 표시한다.

## 검증 기준

- 모바일 폭에서 첫 화면 입력과 CTA가 접히거나 겹치지 않는다.
- 첫 화면에서 10초 멈춤 메시지가 명확히 보인다.
- 6개 메뉴가 과하게 복잡해 보이지 않는다.
- 결과 화면이 퍼센트나 진단처럼 보이지 않는다.
- 문장 복사 액션이 결과 문장 가까이에 있다.
- 보안 모드와 저장 안 함 안내가 사용자가 입력하기 전 인지된다.
